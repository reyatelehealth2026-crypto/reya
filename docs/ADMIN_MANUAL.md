## 2. เข้าสู่ระบบ

### 🌐 URL เข้าใช้งาน
```
https://cny.re-ya.com/auth/login.php
```

### 📝 ขั้นตอนการเข้าสู่ระบบ

1. เปิดเบราว์เซอร์ (แนะนำ Chrome, Edge)
2. พิมพ์ URL ด้านบน
3. ใส่ **Username** และ **Password**
4. กดปุ่ม "เข้าสู่ระบบ"

### 🚪 หากลืมรหัสผ่าน

ติดต่อ Super Admin ให้ reset รหัสผ่านให้

## 3. หน้า Dashboard

### 🎯 Dashboard มีกี่แบบ?

มี **3 แท็บ** ให้เลือกดู:

| แท็บ | ดูอะไรได้ | เหมาะกับใคร |
|------|-----------|-------------|
| **Executive** | ยอดขายรวม กราฟสถิติ | เจ้าของร้าน |
| **CRM** | ลูกค้า แคมเปญ กิจกรรม | ฝ่ายการตลาด |
| **Odoo Overview** | ออเดอร์จาก Odoo เครดิตลูกค้า | ฝ่ายบัญชี |

### 🖥️ วิธีเปลี่ยนแท็บ

```
┌────────────────────────────────────────────────────────┐
│  Dashboard                    [Executive] [CRM] [Odoo] │
└────────────────────────────────────────────────────────┘
                                    ↑
                              คลิกเลือกแท็บ
```

### 📊 ข้อมูลในแต่ละแท็บ

#### Executive Dashboard
- ยอดขายวันนี้ / เดือนนี้ / ปีนี้
- จำนวนออเดอร์
- ลูกค้าใหม่
- กราฟแนวโน้ม

#### CRM Dashboard  
- จำนวนลูกค้าทั้งหมด
- ลูกค้าที่ Active ล่าสุด
- แคมเปญที่กำลังรัน
- ข้อความที่รอตอบ

#### Odoo Overview
- จำนวนลูกค้า Odoo
- ออเดอร์วันนี้ / 30 วัน
- ยอดเงินค้างชำระ (Total Due)
- เครดิตที่ใช้ไป / เครดิตคงเหลือ

## 4. จัดการผู้ใช้ (Users)

### 🎯 หน้านี้ทำอะไรได้?

ดูและจัดการข้อมูลลูกค้า 2 ประเภท:
1. **LINE Users** - ลูกค้าที่ติดตามผ่าน LINE OA
2. **Odoo Customers** - ลูกค้าจากระบบ Odoo ERP

### 🖥️ หน้าตาของหน้า Users

```
┌─────────────────────────────────────────────────────────────┐
│ Users                                           [LINE][Odoo]│
├─────────────────────────────────────────────────────────────┤
│ 🔍 [ค้นหาชื่อ/เบอร์โทร...]    [กรองตามแท็ก ▼]  [ค้นหา]    │
├─────────────────────────────────────────────────────────────┤
│ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐            │
│ │ 👤 สมชาย    │ │ 👤 สมหญิง   │ │ 👤 สมศรี    │            │
│ │ 📱 081xx    │ │ 📱 082xx    │ │ 📱 083xx    │            │
│ │ 🏷️ VIP      │ │ 🏷️ Member   │ │ 🏷️ New      │            │
│ └─────────────┘ └─────────────┘ └─────────────┘            │
└─────────────────────────────────────────────────────────────┘
```

### 🔍 การค้นหาลูกค้า

ค้นหาได้โดยใช้:
- ชื่อที่แสดงใน LINE
- เบอร์โทรศัพท์
- LINE User ID
- ชื่อจริง

### 🏷️ การกรองตามแท็ก (Tag)

แท็กที่มีในระบบ (ตัวอย่าง):
- VIP - ลูกค้าระดับ VIP
- Member - สมาชิกทั่วไป
- New - ลูกค้าใหม่
- Inactive - ไม่ได้ใช้งานนาน

### 📋 รายละเอียดลูกค้า (คลิกที่ชื่อ)

เมื่อคลิกที่ชื่อลูกค้า จะเห็นข้อมูล:
- ข้อมูลส่วนตัว (ชื่อ เบอร์ อีเมล)
- ประวัติการสั่งซื้อ
- แต้มสะสม (Points)
- ระดับสมาชิก (Tier)
- แท็กที่ติดไว้
- ประวัติแชท

## 5. แชทกับลูกค้า (Inbox)

### 🎯 หน้า Inbox เปลี่ยนไปอย่างไร?

**เดิม**: ใช้งานผ่านหน้า `inbox-v2.php` ที่เป็น PHP
**ใหม่**: ระบบจะ **Redirect ไป Next.js Inbox** อัตโนมัติผ่าน SSO

### 🖥️ วิธีเข้าใช้งาน

1. คลิกเมนู "Inbox" ใน Sidebar
2. ระบบจะพาไปยัง `https://inbox.re-ya.com` โดยอัตโนมัติ
3. ไม่ต้อง Login ซ้ำ (SSO)

### 🔄 ถ้าระบบ Next.js มีปัญหา?

ใช้ **Legacy Mode** ได้โดยเพิ่ม `?legacy=1` ที่ท้าย URL:
```
https://cny.re-ya.com/inbox-v2.php?legacy=1
```

### 💡 ฟีเจอร์ใน Inbox

- ดูรายการแชททั้งหมด
- ตอบข้อความลูกค้า
- ดูประวัติการสนทนา
- Assign งานให้เพื่อนร่วมงาน
- ติดแท็กลูกค้า
- ดูข้อมูลลูกค้าข้างๆ หน้าจอ

## 6. จัดการ LINE Accounts

### 🎯 ทำไมต้องตั้งค่า?

ระบบนี้รองรับหลาย LINE Official Account พร้อมกัน ต้องเพิ่มข้อมูลแต่ละบัญชีก่อนใช้งาน

### 📝 ข้อมูลที่ต้องมี

| ข้อมูล | หาได้จากไหน | ตัวอย่าง |
|--------|--------------|----------|
| **Channel ID** | LINE Developer Console | 1234567890 |
| **Channel Secret** | LINE Developer Console | a1b2c3d4e5f6... |
| **Channel Access Token** | LINE Developer Console | BgKx9y... |

### 🖥️ วิธีเพิ่ม LINE Account

1. ไปที่ **Settings** → **LINE Accounts**
2. กดปุ่ม "+ เพิ่มบัญชีใหม่"
3. กรอกข้อมูล:
   - ชื่อบัญชี (ตั้งเองได้)
   - Channel ID
   - Channel Secret
   - Channel Access Token
4. กด "บันทึก"
5. กด "ทดสอบการเชื่อมต่อ" เพื่อยืนยันว่าใช้งานได้

### ⚠️ ข้อควรระวัง

- **Channel Secret** และ **Access Token** เป็นความลับ ห้ามเผยแพร่
- ถ้า Token หมดอายุ ต้องไปสร้างใหม่ที่ LINE Developer Console
- Webhook URL ต้องเป็น HTTPS เท่านั้น

## 7. ข้อมูลจาก Odoo ERP

### 🎯 Odoo คืออะไร?

**Odoo** คือระบบ ERP (Enterprise Resource Planning) ที่ใช้จัดการ:
- บัญชี / การเงิน
- สต็อกสินค้า
- ออเดอร์ / ใบสั่งซื้อ
- ลูกค้า

### 🔌 การเชื่อมต่อ

ระบบ re-ya เชื่อมต่อกับ Odoo ผ่าน:
1. **API** - ดึงข้อมูลออเดอร์ ลูกค้า
2. **Webhook** - Odoo ส่งข้อมูลมาที่ re-ya เมื่อมีการเปลี่ยนแปลง

### 📊 ข้อมูลที่ดึงจาก Odoo

| ข้อมูล | ดูได้ที่ไหน | อัพเดตเมื่อไหร่ |
|--------|-------------|----------------|
| รายการออเดอร์ | Odoo Dashboard | Real-time ผ่าน Webhook |
| สถานะการชำระเงิน | Users → รายละเอียดลูกค้า | Real-time |
| เครดิตลูกค้า | Odoo Dashboard | ทุก 30 นาที |
| ยอดค้างชำระ | Odoo Dashboard | Real-time |

### 🔔 Webhook จาก Odoo

เมื่อมีเหตุการณ์ใน Odoo ระบบจะได้รับข้อมูลทันที:
- สร้างออเดอร์ใหม่
- อัพเดตสถานะการชำระเงิน
- เปลี่ยนแปลงข้อมูลลูกค้า

## 9. การแก้ไขปัญหา (Troubleshooting)

### ❌ ปัญหาที่พบบ่อย

#### 1. เข้า Inbox ไม่ได้ / ขาว

**สาเหตุ:**
- SSO Token หมดอายุ
- Next.js Server ล่ม

**แก้ไข:**
```
1. รีเฟรชหน้าเว็บ (F5)
2. ถ้ายังไม่ได้ ให้เพิ่ม ?legacy=1 ที่ท้าย URL
   https://cny.re-ya.com/inbox-v2.php?legacy=1
```

---

#### 2. ไม่ได้รับข้อความจากลูกค้า

**สาเหตุ:**
- Webhook URL ไม่ถูกต้อง
- Channel Secret ผิด
- SSL Certificate หมดอายุ

**แก้ไข:**
```
1. ไปที่ Settings → LINE Accounts
2. กด "ทดสอบการเชื่อมต่อ"
3. ถ้าไม่ผ่าน → ตรวจสอบ Channel Secret และ Token
4. ไปที่ LINE Developer Console → ตรวจสอบ Webhook URL
   ต้องเป็น: https://cny.re-ya.com/webhook.php?account=1
```

---

#### 3. ข้อมูล Odoo ไม่อัพเดต

**สาเหตุ:**
- Webhook จาก Odoo ไม่มา
- API Key หมดอายุ

**แก้ไข:**
```
1. ไปที่ Odoo Dashboard ดูสถานะ
2. ตรวจสอบว่า webhook URL ถูกต้องไหม:
   https://cny.re-ya.com/api/webhook/odoo.php
3. ตรวจสอบ API Key ใน config/config.php
4. ติดต่อทีม Odoo ให้ตรวจสอบการส่ง Webhook
```

---

#### 4. ส่ง Broadcast ไม่ได้

**สาเหตุ:**
- Token หมดอายุ
- ส่งข้อความเยอะเกิน (Rate Limit)

**แก้ไข:**
```
1. ไปที่ Settings → LINE Accounts → ทดสอบการเชื่อมต่อ
2. ถ้า token หมดอายุ → ไปสร้างใหม่ที่ LINE Developer Console
3. รอ 1 ชั่วโมงแล้วลองใหม่ถ้าโดน Rate Limit
```

---

#### 5. หน้าเว็บโหลดช้า

**แก้ไข:**
```
1. กด F12 → ไปที่แท็บ Network → ดูว่าอะไรโหลดช้า
2. ล้าง Cache (Ctrl+Shift+R)
3. ตรวจสอบว่าเป็นกับทุกหน้าหรือหน้าเดียว
4. ถ้าเป็นหน้า Dashboard → อาจเป็นเพราะดึงข้อมูลเยอะ รอสักครู่
```
