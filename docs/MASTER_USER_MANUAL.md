## 1. ⚙️ เจาะลึกระบบอัตโนมัติ (System Workflows)
ส่วนนี้คือ "สิ่งที่ระบบทำเอง" โดยที่แอดมินไม่ต้องกด เพื่อให้คุณเข้าใจพฤติกรรมของบอท

### 1.1 เมื่อลูกค้า "เพิ่มเพื่อน" (Follow Event)
ทันทีที่มีคน Add Friend ระบบจะทำงานตามลำดับนี้:
1. **สร้างโปรไฟล์ลูกค้า**: ดึงชื่อ/รูปจาก LINE เก็บลงฐานข้อมูลทันที
2. **ติด Tag แรกเข้า**: ลูกค้าใหม่จะถูกติด Tag `New Customer` ให้โดยอัตโนมัติ
3. **เลือก Rich Menu**: ระบบจะตรวจสอบว่าลูกค้าเป็นสมาชิกระดับไหน แล้วแสดง Rich Menu ให้ตรงกลุ่ม (เช่น ถ้าเป็นลูกค้าเก่า จะเห็นเมนู "เช็คแต้ม")
4. **ส่งข้อความต้อนรับ**: ส่ง Welcome Message ที่ตั้งค่าไว้ (พร้อมชื่อลูกค้าและชื่อร้าน)

### 1.2 ระบบ "รอสลิป" (Waiting for Slip State)
**นี่คือฟีเจอร์ลับที่ช่วยลดงานแอดมิน!**
เมื่อมีการสั่งซื้อแบบ "โอนเงิน" สถานะลูกค้าจะเปลี่ยนเป็น `waiting_slip` (รอสลิป)
- **สิงที่เกิดขึ้น**: ถ้าระหว่างนี้ลูกค้าส่ง **"รูปภาพ"** อะไรเข้ามาก็ตาม ระบบจะเข้าใจว่าเป็น **"สลิปโอนเงิน"** ทันที!
- **สิ่งทีระบบทำ**:
  1. บันทึกรูปนั้นเป็นหลักฐานการโอน
  2. เปลี่ยนสถานะออเดอร์เป็น `waiting_proof` (รอตรวจ)
  3. แจ้งเตือนแอดมินทาง Telegram ว่า "มีสลิปเติมเข้ามาใหม่"
  4. ตอบกลับลูกค้าว่า "ได้รับยอดโอนแล้ว กรุณารอสักครู่..."

> **💡 ทริค**: แอดมินไม่ต้องพิมพ์ทวงสลิปซ้ำ ถ้าลูกค้าส่งรูปมา คือจบขั้นตอนทันที

## 2. 💬 ระบบตอบแบบผสมผสาน (Hybrid Chat)

ระบบนี้ใช้ **AI + กฎ (Rules) + คน (Human)** ทำงานร่วมกันดังนี้:

### 2.1 ลำดับการตอบ (Priority)
ระบบจะตรวจสอบข้อความตามลำดับความสำคัญ:
1. **คำสั่งลัด (Slash Command)**: เช่น `/price`, `/stock` -> ระบบตอบทันที (สูงสุด)
2. **คำสั่งระบบ (System Keywords)**: เช่น "menu", "pharma", "stop bot" -> ระบบทำงานตามฟังก์ชัน
3. **AI เภสัชกร**: ถ้าเป็นคำถามสุขภาพ -> AI Gemini จะตอบ
4. **General Auto-Reply**: ถ้าตรงกับ Keyword ที่ตั้งไว้ -> ตอบข้อความอัตโนมัติ
5. **Human**: ถ้าไม่เข้าเงื่อนไขใดๆ -> แอดมินต้องมาตอบเอง

### 2.2 วิธีหยุดบอท (Stop AI)
ถ้าคุยกับลูกค้าไม่รู้เรื่อง หรือ AI ตอบผิด ให้พิมพ์คำสั่งเหล่านี้ (หรือลูกค้าพิมพ์เองก็ได้):
- `stop bot`
- `ขอคุยกับคน`
- `ติดต่อแอดมิน`
- `consult pharmacist`

**ผลลัพธ์**: AI จะหยุดตอบลูกค้ารายนั้นชั่วคราว และแจ้งเตือนเข้า Telegram แอดมินทันที

## 3. 🛍️ ระบบสั่งซื้อและแจ้งโอน (Commerce Flow)

### 3.1 การสั่งผ่านแชท (Conversational Commerce)
ลูกค้าสามารถพิมพ์คำว่า:
- `สั่งซื้อ` หรือ `buy` -> ระบบส่ง Flex Message รายการสินค้า
- `ยืนยัน` หรือ `confirm` -> ใช้ยืนยันออเดอร์ที่ค้างอยู่ (ในกรณีที่สร้างตะกร้าไว้แล้ว)
- `ยกเลิก` หรือ `cancel` -> ลบออเดอร์ที่ค้างอยู่ทิ้ง

### 3.2 การยืนยันสลิป
แอดมินสามารถกดยืนยันสลิปได้ 2 ทาง:
1. **ผ่านหน้า Admin Panel**: เมนู "รายการสั่งซื้อ" > "รอตรวจสลิป"
2. **ผ่าน Telegram**: (ถ้าเปิดใช้งาน) จะมีปุ่ม `Approve` / `Reject` ส่งไปที่กลุ่ม Telegram กดได้เลยไม่ต้องเข้าเว็บ

## 4. 👥 ระบบดูแลลูกค้าอัตโนมัติ (Auto CRM)

### 4.1 Broadcast Click Tracking
เมื่อเราส่ง Broadcast ออกไป ระบบจะแอบติด "ตัวดักจับ" ไว้
- **ถ้าลูกค้ากดรูปสินค้าใน Broadcast**:
  1. ระบบจะนับสถิติ (CTR)
  2. ระบบจะ **ติด Tag** ให้ลูกค้าคนนั้นทันทีตามสินค้าที่กด (เช่น กดดูนมผง -> ติด Tag `สนใจนมผง`)
  3. ครั้งต่อไป เราจะ Broadcast ขายของได้แม่นยำขึ้นจาก Tag นี้

### 4.2 Dynamic Rich Menu
เมนูด้านล่างเปลี่ยนเองได้!
- **ลูกค้าใหม่**: เห็นเมนู "สมัครสมาชิกรับส่วนลด"
- **สมาชิก Silver**: เห็นเมนู "สะสมแต้ม"
- **สมาชิก Gold**: เห็นเมนู "VIP Lounge"
*(ตั้งค่าเงื่อนไขได้ที่เมนู เครื่องมือ > Rich Menu)*

## 5. 🔧 คำสั่งระบบที่พนักงานต้องรู้ (System Commands)

พิมพ์คำเหล่านี้ในช่องแชท (หรือบอกให้ลูกค้าพิมพ์) เพื่อเรียกใช้ฟังก์ชันลัด:

| คำสั่ง (พิมพ์ได้เลย) | ผลลัพธ์ | ใช้เมื่อไหร่ |
|-------------------|---------|--------------|
| `menu`, `help` | เปิดเมนูหลัก (Quick Reply) | ลูกค้าหาเมนูไม่เจอ |
| `shop`, `ร้านค้า` | เปิดหน้าร้าน (LIFF) | อยากซื้อของ |
| `sc`, `score`, `point` | เช็คแต้มสะสม | ลูกค้าถามแต้ม |
| `member`, `บัตร` | เปิดบัตรสมาชิกดิจิทัล | ลูกค้าอยากดูเลขสมาชิก |
| `tracking`, `track` | เช็คเลขพัสดุล่าสุด | ลูกค้าทวงของ |
| `slip`, `โอนเงิน` | เปิดโหมดรับสลิป | ลูกค้าอยากโอนเงิน |
| `@ai [คำถาม]` | บังคับเรียก AI ตอบ | อยากให้ AI ช่วยตอบยา |
