1. ภาพรวมสถาปัตยกรรม

ระบบถูกแบ่งเป็น 4 ส่วนที่ทำงานร่วมกันเหมือนสายพานเดียวกัน:

  1. Odoo ERP – ต้นทางข้อมูลคำสั่งซื้อ ใบแจ้งหนี้ และเครดิต
  2. ศูนย์รับข้อมูล – กลไกที่รับ Webhook แล้วบันทึกลงฐานข้อมูลกลางแบบอัตโนมัติ
  3. หลังบ้าน (Admin Portal) – เมนูต่าง ๆ สำหรับทีมงาน เช่น Dashboard, ร้านค้า, ข้อมูลลูกค้า
  4. LIFF ฝั่งลูกค้า – หน้า LINE ที่ลูกค้าใช้เชื่อมบัญชี ดูสถานะ และส่งหลักฐานการโอน
สิ่งที่ควรรู้: เมื่อ Odoo ส่งข้อมูลเข้ามา ระบบจะเก็บสำรองไว้ในฐานข้อมูลกลางทันที จึงยังสามารถเปิดดูข้อมูลเดิมได้แม้ Odoo ชะลอหรือปิดปรับปรุง

2. บทบาทผู้ใช้และสิทธิ์

แอดมิน (Back Office)

  • เปิดดูยอดรวมที่ หน้า Odoo Dashboard
  • ติดตามคำสั่งซื้อแบบเรียลไทม์จากเมนู Webhook / รายการสั่งซื้อ
  • ตรวจสถานะใบแจ้งหนี้และเครดิต ก่อนอนุมัติการส่งของหรือปล่อยวงเงิน
  • เปิดหน้าแสดงข้อมูลลูกค้า 360° จากเมนูลูกค้าแล้วค้นหาชื่อลูกค้าที่ต้องการ
  • สลับโหมดร้านค้า (ข้อมูลร้านเดิม / Odoo) จากเมนูตั้งค่า

ลูกค้า (ผ่าน LIFF)

  • เชื่อมบัญชี LINE กับข้อมูลลูกค้าใน Odoo ด้วยเบอร์โทรหรือรหัสลูกค้า
  • ตรวจยอดค้างชำระ วงเงิน และสถานะคำสั่งซื้อได้ด้วยตนเอง
  • เลือกเปิดรับการแจ้งเตือน หรือตัดการเชื่อมต่อได้ทุกเวลา

3. คู่มือฝั่งแอดมิน

3.1 หน้า Odoo Dashboard

เปิดที่ odoo-dashboard.php แล้วเลือกแท็บที่ต้องการ

  • บัตรสรุปเครดิต – เห็นวงเงินรวม ใช้ไป และคงเหลือ พร้อมเวลาปรับปรุงล่าสุด
  • ตัวเลขคำสั่งซื้อ – ชี้ว่ามีออเดอร์กี่รายการวันนี้ เดือนนี้ และยอดขายโดยรวม
  • เมนูย่อย 6 ส่วน – สินค้า ลูกค้า คำสั่งซื้อ ใบแจ้งหนี้ ค่าส่ง และ Webhook สามารถคลิกสลับได้ทันที
  • ปุ่มทางลัด – เช่น “สินค้า 20 รายการแรก” กดแล้วระบบจะค้นให้เลย
ทิป: หากยอดไม่ขึ้น ให้กดรีเฟรชหนึ่งครั้ง หรือแจ้งทีมเทคนิคให้ตรวจสอบงานนำเข้าข้อมูลประจำวัน

3.2 หน้าติดตาม Webhook

ใช้ลิงก์ ส่วน Webhook ใน Dashboard หรือหน้าเครื่องมือที่ทีมเตรียมไว้ เพื่อตรวจว่าออเดอร์ได้รับการอัปเดตหรือไม่

  • กล่อง “ภาพรวมวันนี้” แสดงจำนวนเหตุการณ์ที่สำเร็จ ล้มเหลว หรือรอส่งต่อ
  • ตารางด้านล่างให้ค้นหาด้วยชื่อออเดอร์ เลขใบแจ้งหนี้ หรือชื่อผู้รับ
  • กดดูรายละเอียดเพื่อเห็นเส้นทางออเดอร์จากเริ่มต้นจนส่งสำเร็จ

3.3 เมนูร้านค้า (Shop Orders)

เปิดที่ shop/orders.php แล้วสังเกตข้อความด้านบน หากขึ้นว่า “โหมด Odoo” แสดงว่ากำลังอ่านข้อมูลจาก Odoo

  • กล่องค้นหา รองรับทั้งเลขออเดอร์และชื่อลูกค้า
  • ตัวเลือกสถานะฝั่งขวา มีป้ายสี เช่น “ยืนยันการขาย” หรือ “ชำระแล้ว”
  • ตารางจะแสดงลูกค้า ยอดรวม และวันที่ล่าสุดที่ได้รับข้อมูล

3.4 ข้อมูลลูกค้า 360°

เข้าผ่านเมนู ลูกค้า Odoo แล้วกดชื่อลูกค้าที่ต้องการ

  • หากลูกค้าเชื่อมกับ LINE แล้ว จะแสดงข้อมูลส่วนตัว วงเงิน ยอดค้าง และรายการสั่งซื้อล่าสุด
  • ท้ายหน้าเป็นลิสต์ Webhook ล่าสุดเพื่อดูว่าเหตุการณ์ใดเพิ่งเกิดขึ้น
  • หากข้อมูลบางส่วนหายไป ระบบจะแจ้งเตือนให้ทราบพร้อมคำแนะนำ

3.5 ช่วยลูกค้าเชื่อมบัญชี

  1. ยืนยันกับลูกค้าว่ามีบัญชี LINE Official นี้ และมีข้อมูลใน Odoo แล้ว
  2. ให้ลูกค้าเปิดลิงก์ LIFF Odoo Link หรือสแกน QR ในคู่มือ
  3. เลือกวิธีเชื่อม (เบอร์โทร / รหัสลูกค้า / อีเมล) แล้วกดยืนยัน
  4. เมื่อเชื่อมสำเร็จ ให้ลูกค้าทดสอบเปิดหน้า “สถานะเครดิต” หรือ “ออเดอร์ล่าสุด” ใน LIFF

4. ประสบการณ์ฝั่งลูกค้า (LIFF)

ลูกค้าเข้าสู่ระบบผ่านลิงก์ LIFF หรือไอคอนในเมนู LINE โดยหน้าหลักมีส่วนสำคัญดังนี้:

  • เลือกวิธีเชื่อมบัญชี (เบอร์โทร / รหัสลูกค้า / อีเมล) พร้อมข้อความอธิบายชัดเจน
  • เมื่อเชื่อมสำเร็จ ระบบจะแสดงโปรไฟล์ วงเงินคงเหลือ เมนูลัด “ออเดอร์ล่าสุด” และ “สถานะเครดิต”
  • บล็อก “ออเดอร์ล่าสุด” มีป้ายสีบอกสถานะ และคลิกไปหน้ารายละเอียดได้
  • ลูกค้ากดปุ่มเปิด/ปิดการแจ้งเตือนได้จากหน้าหลัก หากปิดไว้จะไม่ได้รับ LINE แจ้งอัปเดต
แอดมินสามารถเปิด หน้า LIFF เวอร์ชันทดลอง บนคอมพิวเตอร์เพื่อสาธิตให้ลูกค้าดูได้ทันที

5. กระบวนการปฏิบัติงานประจำวัน

5.1 เช็กสุขภาพระบบ (Daily)

  1. เปิดหน้า Odoo Dashboard ดูยอดวันนี้ ยอดเดือนนี้ และวันเวลาล่าสุดที่ระบบรับข้อมูล
  2. เลื่อนลงไปส่วน Webhook เพื่อดูจำนวนเหตุการณ์ที่ไม่สำเร็จ หากมีให้แจ้งทีมเทคนิค
  3. เปิด รายการสั่งซื้อ แล้วค้นหาออเดอร์ล่าสุดเพื่อยืนยันว่ามีข้อมูล
  4. เลือกลูกค้า 2 รายจาก เมนูลูกค้า เพื่อตรวจว่ามีคำสั่งซื้อและใบแจ้งหนี้ใหม่หรือไม่

5.2 Onboarding ลูกค้าใหม่

  • สร้างข้อมูลลูกค้าใน Odoo แล้วรอให้ระบบนำเข้ามาในเมนูลูกค้า
  • ส่งลิงก์ LIFF ให้ลูกค้าเชื่อมบัญชี และยืนยันว่าปรากฏวงเงินเรียบร้อย

5.3 การจัดการแจ้งเตือน

หากลูกค้าไม่ได้รับการแจ้งเตือนใน LINE ให้ตรวจตามลำดับนี้:

  • ให้ลูกค้าเปิด LIFF แล้วดูว่าปุ่ม “รับการแจ้งเตือน” เปิดอยู่หรือไม่
  • ในแอดมิน เปิดส่วน Webhook แล้วค้นหาชื่อลูกค้าเพื่อตรวจว่ามีเหตุการณ์ส่งมาหรือไม่
  • ถ้ามีข้อความแจ้งเตือนว่า “ล้มเหลว” ให้ส่งเรื่องต่อทีมเทคนิคเพื่อตรวจการตั้งค่าเซิร์ฟเวอร์

6. การแก้ไขปัญหาและแหล่งข้อมูล

6.1 จุดตรวจสอบหลัก

อาการจุดตรวจสอบการแก้ไข
Dashboard ไม่แสดงยอด กดรีเฟรช และสอบถามทีมเทคนิคว่าระบบอัปเดตข้อมูลล่าสุดเมื่อใด แจ้งทีมเทคนิคให้ตรวจงานนำเข้าข้อมูล
Webhook ติดสถานะ “ล้มเหลว” เปิดส่วน Webhook แล้วกดดูรายละเอียด คัดลอกข้อความแจ้งเตือนส่งให้ทีมเทคนิค
ลูกค้าเชื่อมบัญชีไม่ได้ สอบถามว่ากรอกเบอร์/รหัสถูกต้องหรือไม่ มีข้อความผิดพลาดว่าอะไร ให้ลูกค้าตรวจข้อมูลอีกครั้ง หากยังไม่สำเร็จให้ทีมเทคนิคตรวจสอบ
เมนูร้านค้าไม่พบออเดอร์ ดูว่ามีข้อความ “โหมด Odoo” หรือไม่ และลองค้นหาออเดอร์อื่น หากยังไม่มีข้อมูลให้แจ้งทีมเทคนิคตรวจการรับ Webhook

6.2 Logs สำคัญ

  • บันทึกการเชื่อมต่อ: ให้ทีมเทคนิคตรวจจากฐานข้อมูลหากจำเป็น
  • เว็บฮุค: แอดมินดูได้จากหน้าส่วน Webhook เพื่อทราบว่าติดต่อ Odoo สำเร็จหรือไม่
  • การแจ้งเตือนลูกค้า: หากมีข้อความผิดพลาด ระบบจะแจ้งไว้ท้ายหน้าลูกค้า

7. ภาคผนวก

7.1 โครงสร้างไฟล์เกี่ยวข้อง

  • odoo-dashboard.php – หน้า Dashboard หลักของแอดมิน
  • liff/odoo-link.php – หน้าสำหรับลูกค้าเชื่อมบัญชี
  • shop/orders.php – รายการคำสั่งซื้อโหมด Odoo
  • users.php?tab=odoo – รายชื่อลูกค้าที่ผูก Odoo
