📊 3. หน้า Dashboard

Dashboard คืออะไร?

Dashboard เป็นหน้าสรุปข้อมูลสำคัญทั้งหมดของร้าน ช่วยให้คุณเห็นภาพรวมได้ในหน้าเดียว ไม่ต้องไปคลิกหลายๆ ที่

📑 Dashboard มี 3 แท็บ

แท็บ ดูอะไรได้ เหมาะกับใคร
Executive ยอดขายรวม กราฟสถิติ แนวโน้มธุรกิจ เจ้าของร้าน / ผู้จัดการ
CRM ลูกค้า แคมเปญ กิจกรรม การตลาด ฝ่ายการตลาด / แอดมิน
Odoo Overview ออเดอร์จาก Odoo เครดิตลูกค้า ยอดค้างชำระ ฝ่ายบัญชี / การเงิน
🖥️ วิธีเปลี่ยนแท็บ Dashboard
1
คลิกที่แท็บต้องการ
ที่ด้านบนของหน้า Dashboard จะมีแท็บ [Executive] [CRM] [Odoo] ให้เลือก
2
รอโหลดข้อมูล
ระบบจะดึงข้อมูลล่าสุดมาแสดง ใช้เวลา 1-3 วินาที
3
ดูรายงาน
ข้อมูลจะแสดงในรูปแบบตาราง กราฟ และตัวเลขสรุป

📈 ข้อมูลในแต่ละแท็บ

Executive Dashboard

  • ยอดขายวันนี้ - รายได้จากออเดอร์ที่สร้างวันนี้
  • ยอดขายเดือนนี้ - สรุปรายได้ประจำเดือน
  • จำนวนออเดอร์ - ออเดอร์ทั้งหมด แยกตามสถานะ
  • ลูกค้าใหม่ - จำนวนลูกค้าที่เพิ่มเพื่อนใหม่วันนี้
  • กราฟแนวโน้ม - เปรียบเทียบยอดขายรายวัน/รายเดือน

CRM Dashboard

  • จำนวนลูกค้าทั้งหมด - ยอด Follower ทั้งหมด
  • ลูกค้าที่ Active - คนที่พึ่งมีกิจกรรมล่าสุด
  • แคมเปญที่กำลังรัน - Broadcast ที่ตั้งเวลาไว้
  • ข้อความที่รอตอบ - แชทที่ยังไม่ได้ตอบกลับ

Odoo Overview

  • จำนวนลูกค้า Odoo - ลูกค้าที่มีบัญชีใน ERP
  • ออเดอร์วันนี้ - ออเดอร์จาก Odoo วันนี้
  • ยอดค้างชำระ (Total Due) - เงินที่ลูกค้ายังไม่จ่าย
  • เครดิตที่ใช้ไป/คงเหลือ - วงเงินเครดิตลูกค้า

ทริค: ให้เปิด Executive Dashboard ทิ้งไว้เป็นหน้าแรกตอนเริ่มงาน จะได้เห็นยอดขายวันนี้ทันทีที่ Login

👥 4. จัดการผู้ใช้ (Users)

หน้า Users ใช้ทำอะไร?

หน้านี้เป็นฐานข้อมูลลูกค้าทั้งหมด แบ่งเป็น 2 ประเภทหลัก: LINE Users (ลูกค้าที่เพิ่มเพื่อน LINE OA) และ Odoo Customers (ลูกค้าที่มีบัญชีในระบบ ERP)

👤 ประเภทของ Users

ประเภท แหล่งที่มา ข้อมูลที่มี
LINE Users จากการเพิ่มเพื่อน LINE OA ชLINE, รูปโปรไฟล์, เบอร์โทร (ถ้ามี)
Odoo Customers จากระบบ Odoo ERP ชื่อ-นามสกุล, ที่อยู่, เครดิต, ยอดค้าง
🔍 วิธีค้นหาลูกค้า
1
พิมพ์คำค้นหา
ใส่ในช่อง Search ด้านบน - ค้นหาได้จากชื่อ, เบอร์โทร, หรือ LINE ID
2
เลือก Filter (ถ้าต้องการ)
กรองตามแท็ก เช่น VIP, Member, New, Inactive
3
กดค้นหา
ระบบจะแสดงรายชื่อที่ตรงกับเงื่อนไข
4
คลิกดูรายละเอียด
กดที่ชื่อลูกค้าเพื่อดูข้อมูลเต็ม

🏷️ แท็ก (Tags) ที่มีในระบบ

แท็ก ความหมาย ใช้เมื่อไหร่
VIP ลูกค้าระดับ VIP ลูกค้าซื้อบ่อย หรือมียอดสูง
Member สมาชิกทั่วไป ลูกค้าที่ลงทะเบียนแล้ว
New ลูกค้าใหม่ เพิ่งเพิ่มเพื่อนใน 7 วัน
Inactive ไม่ได้ใช้งานนาน ไม่มีกิจกรรม 30+ วัน
📋 ข้อมูลในรายละเอียดลูกค้า
1
ข้อมูลส่วนตัว
ชื่อจริง, เบอร์โทร, อีเมล, ที่อยู่จัดส่ง
2
ประวัติการสั่งซื้อ
รายการออเดอร์ทั้งหมด สถานะ ยอดเงิน
3
แต้มสะสม (Points)
ยอดคะแนนปัจจุบัน และประวัติการได้/ใช้แต้ม
4
ระดับสมาชิก (Tier)
Bronze, Silver, Gold, Platinum เป็นต้น
5
ประวัติแชท
ดูการสนทนาล่าสุดกับลูกค้า

ทริค: ใช้ Filter "Inactive" แล้วส่ง Broadcast โปรโมชั่นพิเศษไปกระตุ้นให้กลับมาซื้ออีก

💬 5. แชทกับลูกค้า (Inbox)

Inbox มีอะไรใหม่?

ระบบ Inbox ได้อัพเกรดเป็น Next.js Inbox ที่เร็วขึ้น สวยงามขึ้น แต่ถ้าเจอปัญหาสามารถใช้ Legacy Mode ได้

🔄 ความแตกต่างระหว่างโหมด

โหมดปกติ (Next.js): ใช้งานผ่าน https://inbox.re-ya.com เร็ว ลื่น มีฟีเจอร์ครบ

Legacy Mode: ใช้งานผ่าน PHP เดิม สำรองไว้กรณี Next.js มีปัญหา

🖥️ วิธีเข้าใช้งาน Inbox
1
คลิกเมนู "Inbox"
ใน Sidebar ทางซ้ายมือ
2
รอ Redirect
ระบบจะพาไปยัง https://inbox.re-ya.com อัตโนมัติผ่าน SSO
3
เริ่มใช้งาน
ไม่ต้อง Login ซ้ำ ระบบจะจดจำการ Login จากหน้าหลัก

💡 ฟีเจอร์หลักใน Inbox

ฟีเจอร์ ใช้ทำอะไร ทางลัด
รายการแชท ดูรายชื่อลูกค้าที่คุยด้วย เรียงตามล่าสุด -
ตอบข้อความ พิมพ์ตอบกลับลูกค้าแบบ Real-time Enter
Assign มอบหมายงานให้เพื่อนร่วมงาน Ctrl+A
ติดแท็ก ติด Tag ให้ลูกค้ารายนี้ Ctrl+T
ดูโปรไฟล์ ดูข้อมูลลูกค้าข้างๆ หน้าจอ Ctrl+P
Quick Reply ใช้ข้อความที่ตั้งไว้ล่วงหน้า / + ชื่อ template

⚠️ ถ้าเข้า Inbox ไม่ได้ (หน้าขาว/Error)

  1. ลอง รีเฟรชหน้า (F5)
  2. ถ้ายังไม่ได้ ให้เพิ่ม ?legacy=1 ที่ท้าย URL
  3. URL ที่ใช้: https://cny.re-ya.com/inbox-v2.php?legacy=1

ทริคการตอบลูกค้าเร็ว: สร้าง Quick Reply Templates ไว้ล่วงหน้า เช่น "สวัสดีค่ะ ยินดีให้บริการค่ะ", "ขอบคุณที่ใช้บริการค่ะ" แล้วใช้ / ตามด้วยชื่อ template

💚 6. จัดการ LINE Accounts

ทำไมต้องตั้งค่า LINE Accounts?

ระบบนี้รองรับหลาย LINE Official Account พร้อมกัน ต้องเพิ่มข้อมูลแต่ละบัญชีก่อนถึงจะรับส่งข้อความได้

📝 ข้อมูลที่ต้องมีจาก LINE Developer

ข้อมูล หาได้จากไหน ความสำคัญ
Channel ID Basic settings tab ระบุตัวตนของบัญชี
Channel Secret Basic settings tab 🔐 ความลับสูง ใช้ Verify webhook
Channel Access Token Messaging API tab 🔐 ใช้ส่งข้อความ
🖥️ วิธีเพิ่ม LINE Account
1
ไปที่ Settings
คลิก "Settings" ใน Sidebar → เลือกแท็บ "LINE Accounts"
2
กด "+ เพิ่มบัญชีใหม่"
ปุ่มสีเขียวมุมขวาบน
3
กรอกข้อมูล
ชื่อบัญชี (ตั้งเองได้), Channel ID, Secret, Token
4
กด "บันทึก"
ระบบจะบันทึกข้อมูลลง Database
5
ทดสอบการเชื่อมต่อ
กดปุ่ม "ทดสอบ" เพื่อยืนยันว่าใช้งานได้จริง

🔐 ข้อควรระวังด้านความปลอดภัย

  • Channel Secret และ Access Token เป็นความลับ ห้ามเผยแพร่เด็ดขาด
  • ถ้า Token หมดอายุ ต้องไปสร้างใหม่ที่ LINE Developer Console
  • Webhook URL ต้องเป็น HTTPS เท่านั้น (ไม่รองรับ HTTP)
  • อย่าแชร์ข้อมูล Login กับคนอื่น

🔗 การตั้งค่า Webhook ใน LINE Developer

# URL ที่ต้องใส่ใน LINE Developer Console: https://cny.re-ya.com/webhook.php?account=1 # account=1 คือ ID ของบัญชีแรก ถ้ามีหลายบัญชีจะเป็น 2, 3, ...
การเปิดใช้งาน Webhook
1
เข้า LINE Developer Console
https://developers.line.biz/console/
2
เลือก Provider และ Channel
เข้าไปที่ Messaging API Channel ของคุณ
3
ตั้งค่า Webhook URL
ใส่ URL ด้านบนในช่อง Webhook URL
4
เปิดใช้งาน
กด "Verify" แล้วเปิดสวิตช์ "Use webhook"
5
ปิด Auto-reply
ใน Messaging settings ให้ปิด Auto-reply เพื่อให้ Webhook ทำงาน

📦 7. ข้อมูลจาก Odoo ERP

Odoo คืออะไร?

Odoo คือระบบ ERP (Enterprise Resource Planning) ที่ใช้จัดการบัญชี สต็อกสินค้า ออเดอร์ และลูกค้า ระบบ re-ya เชื่อมต่อกับ Odoo เพื่อดึงข้อมูลมาแสดงในหน้าเดียว

🔌 การเชื่อมต่อ 2 ทาง

ทิศทาง วิธีการ ข้อมูลที่ส่ง
Odoo → re-ya Webhook (Real-time) ออเดอร์ใหม่, การชำระเงิน, อัพเดตสถานะ
re-ya → Odoo API Request ดึงข้อมูลลูกค้า, เช็คเครดิต, ออเดอร์

📊 ข้อมูลที่ดึงจาก Odoo

ข้อมูล ดูได้ที่ไหน อัพเดตเมื่อไหร่
รายการออเดอร์ Odoo Dashboard Real-time ผ่าน Webhook
สถานะการชำระเงิน Users → รายละเอียดลูกค้า Real-time
เครดิตลูกค้า Odoo Dashboard ทุก 30 นาที
ยอดค้างชำระ Odoo Dashboard Real-time
ที่อยู่ลูกค้า User Detail เมื่อมีการอัพเดต
🔔 Webhook Events จาก Odoo
1
สร้างออเดอร์ใหม่
เมื่อมีการสร้าง Sales Order ใน Odoo ระบบจะแจ้งมาทันที
2
อัพเดตสถานะการชำระเงิน
เมื่อลูกค้าชำระเงิน หรือมีการยืนยันสลิป
3
เปลี่ยนแปลงข้อมูลลูกค้า
เมื่อมีการแก้ไขข้อมูลใน Odoo
4
อัพเดตเครดิต
เมื่อวงเงินเครดิตมีการเปลี่ยนแปลง

ทริค: ถ้าลูกค้าถามเรื่องยอดค้างชำระ ให้ไปดูที่ User Detail → แท็บ Odoo จะเห็นข้อมูลล่าสุดจากระบบบัญชีทันที ไม่ต้องโทรไปถามแผนกบัญชี

⚙️ 8. การตั้งค่าระบบ

🔄 9. Workflow ประจำวัน

Checklist สำหรับแอดมินแต่ละกะ

ทำตาม Checklist นี้ทุกวัน เพื่อให้ระบบทำงานได้อย่างเต็มประสิทธิภาพ

☀️ Checklist กะเช้า (เปิดร้าน)

🛠️ 10. การแก้ไขปัญหา

❌ ปัญหาที่พบบ่อยและวิธีแก้

1. เข้า Inbox ไม่ได้ / หน้าขาว

สาเหตุ: SSO Token หมดอายุ หรือ Next.js Server มีปัญหา

แก้ไข:

  1. รีเฟรชหน้าเว็บ (F5)
  2. ถ้ายังไม่ได้ ใช้ Legacy Mode:
    https://cny.re-ya.com/inbox-v2.php?legacy=1
  3. ถ้ายังไม่ได้ ติดต่อ IT Support

2. ไม่ได้รับข้อความจากลูกค้า

สาเหตุ: Webhook URL ไม่ถูกต้อง หรือ Channel Secret ผิด

แก้ไข:

  1. ไปที่ Settings → LINE Accounts
  2. กด "ทดสอบการเชื่อมต่อ"
  3. ถ้าไม่ผ่าน → ตรวจสอบ Channel Secret และ Token
  4. ไปที่ LINE Developer Console
  5. ตรวจสอบ Webhook URL ต้องเป็น:
    https://cny.re-ya.com/webhook.php?account=1
  6. ตรวจสอบว่าเปิด "Use webhook" และปิด "Auto-reply"

3. ข้อมูล Odoo ไม่อัพเดต

สาเหตุ: Webhook จาก Odoo ไม่มา หรือ API Key หมดอายุ

แก้ไข:

  1. ไปที่ Odoo Dashboard ดูสถานะ
  2. ตรวจสอบว่า Webhook URL ถูกต้อง:
    https://cny.re-ya.com/api/webhook/odoo.php
  3. ตรวจสอบ API Key ใน config (ต้องให้ Dev ช่วย)
  4. ติดต่อทีม Odoo ให้ตรวจสอบการส่ง Webhook

4. ส่ง Broadcast ไม่ได้

สาเหตุ: Token หมดอายุ หรือส่งข้อความเยอะเกิน (Rate Limit)

แก้ไข:

  1. ไปที่ Settings → LINE Accounts → ทดสอบการเชื่อมต่อ
  2. ถ้า Token หมดอายุ → ไปสร้างใหม่ที่ LINE Developer Console
  3. ถ้าโดน Rate Limit → รอ 1 ชั่วโมงแล้วลองใหม่
  4. ตรวจสอบ Quota การสBroadcast รายเดือน

5. หน้าเว็บโหลดช้า

แก้ไข:

  1. กด F12 → ไปที่แท็บ Network → ดูว่าอะไรโหลดช้า
  2. ล้าง Cache (Ctrl+Shift+R)
  3. ตรวจสอบว่าเป็นกับทุกหน้าหรือหน้าเดียว
  4. ถ้าเป็นหน้า Dashboard → อาจเป็นเพราะดึงข้อมูลเยอะ รอสักครู่
  5. ลองเปิดด้วยเบราว์เซอร์อื่น
  6. เช็คความเร็วอินเทอร์เน็ต
