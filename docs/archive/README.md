# เอกสาร Archive (Legacy / Historical)

โฟลเดอร์นี้ใช้สำหรับรวบรวมและแยกเอกสารเก่า/เอกสารสรุปรายงานรายงานงานย่อย (historical docs) ออกจากเอกสารหลักที่ใช้ปฏิบัติการปัจจุบัน

## วัตถุประสงค์
- ลดความสับสนระหว่างเอกสารปัจจุบันกับเอกสารเก่า
- เก็บประวัติการตัดสินใจและผลทดสอบย้อนหลัง
- ทำให้ onboarding ทีมใหม่ง่ายขึ้น

## โครงสร้างที่ใช้
- `LEGACY_DOC_CATALOG_2026-02.md` : ดัชนีเอกสารเก่าและกลุ่มเอกสาร

## ข้อกำหนดการใช้งาน
1. เอกสารที่ active ปัจจุบันให้เก็บไว้ที่ `docs/` ระดับหลัก
2. เอกสารที่เป็น task summary, quick fix, incident report, test report รุ่นเก่า ให้บันทึกใน catalog นี้
3. หากมีการย้ายไฟล์จริงในอนาคต ให้คง redirect/index ไว้เสมอเพื่อไม่ให้ลิงก์ภายในแตก
