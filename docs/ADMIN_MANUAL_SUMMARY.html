01 · ภาพรวมสถาปัตยกรรม & ฟีเจอร์หลัก

ระบบ re-ya เป็นศูนย์กลาง (hub) ที่ดึงข้อมูลจาก LINE Official Account และ Odoo ERP แล้วส่งต่อให้ทีมงานเห็นภาพเดียวกันทั้งการตลาด, การขาย, บัญชี และเภสัชกรรม

ฟีเจอร์หลัก รายละเอียด กลุ่มผู้ใช้
Dashboard แท็บ Executive/CRM/Odoo แสดงยอดขาย, กิจกรรมลูกค้า, เครดิต, KPI Owner · Marketing · Finance
Next.js Inbox ตอบโต้แชท, Assign, Quick Reply, Tagging, Legacy fallback CRM Agent · Pharmacist
Users 360° ผสานข้อมูล LINE + Odoo, แต้มสะสม, Tier, ประวัติสั่งซื้อ Admin · Sales · Accounting
Integration Hub LINE OA, Odoo ERP, Telegram, Email, AI Chat Technical Admin
จุดแข็งสำคัญ
  • SSO เชื่อมต่อ Next.js Inbox แบบ Single Click
  • Webhook Processor มี Failover & Logging แยก
  • Legacy PHP Inbox standby ด้วย parameter `?legacy=1`
  • เชื่อมโอเพ่นแชนแนล LINE หลายบัญชีพร้อมกัน

02 · Dashboard & Analytics

Dashboard รวมข้อมูลที่จำเป็นสำหรับทั้งผู้บริหารและทีมปฏิบัติการ โดยแบ่งเป็น 3 แท็บเพื่อการวิเคราะห์ที่ชัดเจน

แท็บ ดูอะไรได้ Insight ที่ได้
Executive ยอดขายรายวัน/เดือน/ปี, จำนวนออเดอร์, ลูกค้าใหม่, กราฟแนวโน้ม วัดผลงานภาพรวมธุรกิจและเทียบกับแผนรายเดือน
CRM จำนวนลูกค้า, Active customer, แคมเปญที่รันอยู่, ข้อความรอตอบ จัดลำดับงานทีมตอบแชทและวัดคุณภาพแคมเปญ
Odoo Overview จำนวนลูกค้า Odoo, ออเดอร์วันนี้/30 วัน, ยอดค้างชำระ, เครดิตคงเหลือ จับตาสถานะการเงินและวงเงินของลูกค้าองค์กร
วิธีใช้งานเร็ว

คลิกแท็บที่ต้องการ → รอโหลด 1-3 วินาที → ใช้ Hover เพื่อดูรายละเอียด → Export ภาพหน้าจอส่งผู้บริหารได้ทันที

03 · Users & Next.js Inbox

Users 360°

ฐานข้อมูลลูกค้าทั้งจาก LINE OA และ Odoo ถูกผสานในหน้าจอเดียว ค้นหาด้วยชื่อ, เบอร์, LINE ID หรือแท็กได้

การค้นหา

วิธีค้น: พิมพ์คำค้น → เลือก Filter (VIP/New/Inactive) → ดูผลลัพธ์ทันที

ข้อมูลที่เห็น

ข้อมูลส่วนตัว, ประวัติคำสั่งซื้อ, แต้มสะสม, Tier, ประวัติแชท

Action ต่อเนื่อง

ส่งข้อความ, ติดแท็ก, เปิดประวัติ Odoo, เรียกดูเครดิตคงเหลือ

Next.js Inbox

ศูนย์กลางตอบลูกค้าแบบ Real-time รองรับการ Redirect อัตโนมัติและมีโหมด Legacy สำรอง

ฟีเจอร์เด่น
  • SSO → เข้าถึงหน้า Next.js โดยไม่ต้อง Login ซ้ำ
  • Quick Reply, Assign, Tag, ดูโปรไฟล์ข้างจอ
  • Legacy Mode: `https://cny.re-ya.com/inbox-v2.php?legacy=1`
  • Shortcut สำคัญ: Enter ส่งข้อความ, Ctrl+A Assign, Ctrl+T Tag

04 · LINE Accounts & Odoo Data

จัดการ LINE OA หลายบัญชี

รองรับ Messaging API หลายบัญชีในองค์กร ต้องเตรียม Channel ID, Secret, Access Token ให้ครบ

ขั้นตอนเพิ่มบัญชี

Settings → LINE Accounts → + เพิ่มบัญชี → กรอกข้อมูล → บันทึก → ทดสอบ

Webhook URL

`https://cny.re-ya.com/webhook.php?account={id}` → Verify และเปิด Use Webhook

ความปลอดภัย

เก็บ Secret/Tokens อย่างปลอดภัย, ใช้ HTTPS, ปิด Auto-reply ใน LINE

ข้อมูลจาก Odoo

ข้อมูลบัญชีและเครดิตจาก Odoo ถูกนำมาแสดงใน Dashboard, หน้า Users และรายงานเครดิต

ข้อมูล ตำแหน่งในระบบ ช่องทางอัปเดต
Sales Orders Odoo Dashboard / Users Webhook Real-time
Payment & Residual Users → Odoo Tab Webhook + API Fallback
Customer Credit Odoo Overview Sync ทุก 30 นาที
Address & Profile User Detail Webhook เมื่อมีการแก้ไข
Tip

หาก residual ไม่ถูกส่งมาจาก Odoo ใช้ amount_total เป็น fallback เพื่อให้ข้อมูลเครดิตยังครบ

05 · Settings, Automation & AI

หน้า Settings รวมทุกการตั้งค่าตั้งแต่บัญชี, ข้อความตอบกลับ, LIFF, AI Chat, Telegram, Email/SMTP และ Consent

แท็บ ใช้สำหรับ ความถี่/ผู้รับผิดชอบ
LINE Accounts เพิ่ม/แก้ข้อมูล Messaging API, ทดสอบการเชื่อมต่อ ครั้งแรก + เมื่อ Token เปลี่ยน (Admin)
ข้อความต้อนรับ ตั้ง Auto-reply พร้อม Emoji/โปรโมชั่น อัปเดตทุกแคมเปญ (Marketing)
LIFF & AI กำหนด LIFF URL, ปรับ Prompt AI Dev/AI Lead
Notifications แจ้งเตือนข้อความใหม่, ออเดอร์, สลิป Admin
Consent ข้อความ PDPA, บันทึกการยินยอม Compliance
แนวปฏิบัติ

บันทึกข้อความต้อนรับเป็นเทมเพลต + ใช้ Emoji เพื่อเพิ่มยอดการโต้ตอบ และตั้ง Reminder ตรวจสอบ Token ทุกไตรมาส

06 · Workflow ประจำวัน

กะเช้า (เปิดร้าน)

Login → ตรวจ Dashboard → เช็ค Odoo Dashboard → เปิด Inbox → ทดสอบ Webhook

ระหว่างวัน

ตอบแชทภายใน 5 นาที, ตรวจ Odoo ทุก 30 นาที, อัปเดตสถานะออเดอร์, ตามสลิปการโอน

กะดึก (ปิดร้าน)

สรุปยอดขายวันเดียวกัน, ส่งต่อออเดอร์ค้าง, ตอบข้อความคงค้าง, ส่งรายงานผู้จัดการ

Checklists

  • ตรวจ Inbox ทุกชั่วโมง
  • ยืนยัน Webhook เดือนละครั้ง
  • สำรองข้อมูลรายสัปดาห์

Order Pipeline

Inbox รับคำสั่ง → ตรวจสต็อก → สร้างออเดอร์ Odoo → ส่ง Flex แจ้งโอนเงิน → ตรวจสลิป → จัดส่ง → อัปเดตสถานะ

Broadcast Routine

เลือก Segment จากแท็ก → สร้างข้อความ + ภาพ → ตั้งเวลา → Monitor การส่ง & Reply

07 · Troubleshooting & Support

ปัญหา สาเหตุหลัก แนวทางแก้ไข
เข้า Inbox ไม่ได้ / หน้าขาว SSO Token หมดอายุ, Next.js มีปัญหา รีเฟรช → เปิด Legacy Mode `?legacy=1` → ติดต่อ IT ถ้ายังไม่หาย
ไม่รับข้อความจากลูกค้า Webhook URL ผิด, Secret/Token ใช้ไม่ได้ Settings → LINE Accounts → ทดสอบ → แก้ URL ให้เป็น HTTPS ที่ถูกต้อง
ข้อมูล Odoo ไม่อัปเดต Webhook ล้มเหลว, API Key หมดอายุ ตรวจ Dashboard, ตรวจ URL `api/webhook/odoo.php`, รีเจน API Key
ส่ง Broadcast ไม่ได้ Token หมดอายุ, เกิน Rate Limit สร้าง Token ใหม่, รอ 1 ชั่วโมงแล้วลองอีกครั้ง
หน้าโหลดช้า ดึงข้อมูลจำนวนมากหรือ cache ค้าง F12 → Network → ตรวจ slow asset, Ctrl+Shift+R, ลองเบราว์เซอร์อื่น
ช่องทางติดต่อ
LINE / Inbox ทีมพัฒนา · LINE: @support · dev@re-ya.com
Odoo / บัญชี ทีม Odoo · odoo@cny.co
บัญชีผู้ใช้ Super Admin · LINE: @admin
แจ้ง Bug GitHub Issues: github.com/re-ya/crm/issues
